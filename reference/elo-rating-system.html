ELO Rating System — Quick Reference

Core Formula

Expected outcome:
E_A = 1 / (1 + 10^((R_B − R_A) / 400))

Rating update:
R_A_new = R_A + K × (S_A − E_A)

Where:
• R_A, R_B = current ratings of teams A and B
• S_A = actual score (1 = win, 0.5 = draw, 0 = loss)
• K = weight factor (see table below)

K-Factor Table (for this project)

TournamentK-FactorRationale
FIFA World Cup60Highest stakes, most meaningful results
Continental (Euros, Copa America)50High stakes, strong teams
World Cup Qualifiers40Competitive but uneven matchups
Friendlies20Teams experiment, less meaningful

Worked Example

Brazil (ELO 1800) vs Switzerland (ELO 1650), World Cup match

1. Expected: E_brazil = 1 / (1 + 10^((1650-1800)/400)) = 1 / (1 + 10^(-0.375)) = 0.704
2. Switzerland wins! S_brazil = 0, K = 60
3. Update: R_brazil = 1800 + 60 × (0 − 0.704) = 1800 − 42.2 = 1757.8
4. Update: R_swiss = 1650 + 60 × (1 − 0.296) = 1650 + 42.2 = 1692.2

Note: ELO is zero-sum — points gained by one team = points lost by the other.

Key Properties

  • Self-correcting: a team rated too high will lose ELO when upset; too low will gain ELO
  • Zero-sum: total ELO across all teams stays constant
  • 400-point difference: team with +400 ELO is expected to win ~91% of the time
  • K-factor trade-off: high K = reactive to recent results; low K = stable over time
⚠ Data Leakage Alert: When computing ELO for a match on date D, you must only use match results from BEFORE date D. Process matches in chronological order.
