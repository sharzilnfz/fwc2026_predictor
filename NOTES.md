# Working notes for the teaching workspace

## Learner Profile
- Intermediate Python skills — comfortable with pandas/sklearn from tutorials but no end-to-end project
- Wants to learn by doing — provide guidance, context, and plans, NOT finished code
- Motivated by portfolio value — cares about presentation, documentation, and explainability
- Time-sensitive — 2026 World Cup is live, which creates urgency and excitement

## Teaching Preferences
- Prefers structured learning paths with clear milestones
- Wants to understand WHY each decision is made, not just WHAT to do
- Learn-by-doing: give skeleton code with TODOs, not complete implementations
- Values: seeing results early and iterating, rather than perfecting before seeing output

## Key Risks to Watch
- Scope creep: may want to add player-level features, live scraping, etc. Keep them focused on the core pipeline first.
- Perfectionism: "I need to understand everything before I start." Push them to ship a baseline and iterate.
- Feature engineering rabbit holes: easy to spend days on features that don't move the needle. Guide toward high-impact features first.
- Dashboard polish before model quality: the model is the portfolio differentiator, not the dashboard.
